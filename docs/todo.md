# NPU边缘推理演示项目 - 开发计划

## 项目背景

**目标**: 基于飞腾开发板，部署小型目标检测模型，实现静态图片推理，对外提供API服务

**现状**:
- 硬件: 飞腾派开发板 x1 (无外设)
- 资料: 百度网盘200G资料库 (需筛选)
- 目标: 边缘推理盒子，API调用方式

## 任务一：Agent知识库构建

### 1.1 资料筛选与获取

**目标**: 从200G资料中筛选出有用部分，建立可检索知识库

**步骤**:
1. 梳理飞腾派开发板的资料目录结构
2. 制定下载清单，按需获取
3. 建立本地资料库，建立索引 (按主题/关键词分类)

**交付物**: 整理好的资料库

### 1.2 知识库建设（两种方案）

#### 方案一：直接检索的文本知识库
**目标**: 将文档转化为Agent可检索的格式，检索原则是渐进式披露，通过目录和索引进行逐级展开，最终能定位到具体的知识条目。
**步骤**:
1. 设定知识条目格式
2. 设计知识库结构
3. 切片文档，提取核心信息，形成知识条目
4. 建立关键词索引，便于Agent检索
**交付物**:
- 文本知识库
- 知识库索引
- 检索接口说明

#### 方案二：基于向量数据库的知识库
**目标**: 将文档内容转化为向量，利用向量数据库进行语义检索。
**步骤**:
1. 选择合适的向量数据库（如FAISS、Pinecone等）
2. 设计向量化流程（文本预处理、分词、向量化）
3. 将文档内容切片，转化为向量并存储到数据库中
4. 实现语义检索接口，支持Agent根据查询语句获取相关知识
**交付物**:
- 向量数据库实例
- 向量化处理代码
- 语义检索接口说明

### 1.3 检索逻辑完善

**目标**: 让Agent能准确找到需要的知识

**步骤**:
1. 评估当前检索效果 (关键词匹配、相关性排序)
2. 优化检索策略 (同义词扩展、权重调整)
3. 添加知识关联 (相关条目链接、上下文补全)
4. 测试检索准确率，持续迭代

**交付物**: 检索逻辑优化记录 + 测试用例

## 任务二：板子编译部署链路

### 2.1 开发环境准备

**目标**: 建立可用的交叉编译环境

**步骤**:
1. 确认开发板型号和CPU架构
2. 获取或搭建交叉编译工具链
3. 验证工具链可用性 (编译简单程序测试)
4. 整理环境搭建文档

**交付物**: 可复现的环境搭建指南

### 2.2 板子通信链路

**目标**: 实现主机到板子的文件传输和命令执行

**步骤**:
1. 确定通信方式
2. 建立连接，验证通路
3. 实现文件传输
4. 实现远程命令执行

**交付物**: 通信链路使用指南

### 2.3 编译部署验证

**目标**: 把代码跑通在板子上

**步骤**:
1. 选一个简单程序作为Hello World
2. 交叉编译生成ARM64可执行文件
3. 部署到板子并运行验证
4. 记录完整流程，形成标准操作步骤

**交付物**: 编译部署标准流程文档

### 2.4 Agent Tool对接

**目标**: 让Agent可以自动执行上述流程

**步骤**:
1. 设计Tool接口 (编译 / 部署 / 执行)
2. 实现Tool逻辑 (调用本地交叉编译器，SCP部署，SSH执行)
3. 与AgentCore集成
4. 端到端测试: "在板子上运行XXX" → Agent自动完成编译部署执行

**交付物**: Agent Tool代码 + 使用示例
